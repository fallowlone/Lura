# Дорожная карта: графическая модель (PDF-подобные возможности)

**Статус:** в работе — **фазы B и C (MVP)** реализованы в движке; **фаза A** и всё ниже C — по-прежнему план.  
**Связь:** см. [PROGRESS.md](../PROGRESS.md) — раздел «Графика и композитинг».

Цель: чтобы формат (планируемое имя **Lura**, расширение `**.lura`**) мог выражать и экспортировать возможности уровня зрелых PDF-рендереров: **встроенные шрифты**, **прозрачность и группы**, **маски и clip**, без привязки к одному только плоскому списку примитивов.

---

## Текущая архитектура (IR)

В `DrawCommand` / `PainterCommand` добавлен **стек графического состояния** (подмножество):

- `PushOpacity` / `PopOpacity` — группа с постоянной альфой до `Pop`.
- `PushClipRect` / `PopClip` — прямоугольный clip в координатах страницы.

Пагинация вставляет эти команды вокруг контента блока по `ResolvedStyles` (см. фазы B–C). Для **изолированных групп**, **blend modes**, **SMask** по-прежнему нужны расширения IR и отдельный lowering (фазы D–E).

## Фазы

### Фаза A — Нестандартные шрифты (фундамент)

**Формат:** ресурс шрифта в ассетах (inline base64 или linked + hash); в стилях ссылка на этот ресурс + weight/style.

**Движок:** подстановка байтов шрифта в `fontdb`/shaping; **PDF:** встраивание subset TTF/OTF через `pdf-writer` (обвязка вручную); **SVG:** `@font-face` / embedded font; **preview (WGPU):** та же байтовая загрузка, что и для PDF.

**Критерий готовности:** один и тот же документ с кастомным шрифтом даёт корректный текст в PDF, SVG и PNG preview.

---

### Фаза B — Простая прозрачность — **сделано (MVP)**

**Формат:** атрибут блока `opacity` (0…1) в `ResolvedStyles`; в AST — через attrs, **не наследуется** (как отдельное свойство блока).

**Пайплайн:** `paginate` оборачивает фон + контент блока в `PushOpacity` / `PopOpacity` (порядок: opacity → фон → при необходимости clip → контент).

**Рендер:**

- **PDF:** `q` / `Q` + `ExtGState` с `ca` и `CA` на каждое уникальное значение альфы на странице.
- **SVG:** вложенный `<g opacity="…">`.
- **WGPU (`wgpu-preview`):** альфа в вершинах/цвете текста, `BlendState::ALPHA_BLENDING`, батчи сбрасываются при смене opacity/clip.

**Ограничения:** вложенная композиция без isolated groups — как в плане фазы B (`ITEM` проходит через те же `PushOpacity` / `PopOpacity`, что и обычный текст).

**Критерий (оставшийся долг):** жёсткие golden / визуальное сравнение перекрывающихся блоков между PDF, SVG и PNG.

---

### Фаза C — Clip (обрезка) — **MVP сделано**

**Формат:** `overflow: clip` или `overflow: hidden` → `overflow_clip` в стилях; область клипа = **оценка высоты блока** `estimate_height` (та же, что для фона), прямоугольник в координатах страницы.

**Пайплайн:** после фона блока — `PushClipRect` / после контента — `PopClip` (внутри группы opacity, если она есть).

**Рендер:**

- **PDF:** `q`, путь-прямоугольник, `W n`, затем рисование, `Q`.
- **SVG:** `<defs><clipPath id="…">` + `<g clip-path="url(#…)">`.
- **WGPU:** вложенный **scissor** (пересечение прямоугольников), батчи по смене состояния.

**Ограничения:** при **разрыве блока на несколько страниц** один прямоугольник по оценке высоты даёт неточный clip; контур/путь — **не** в этом MVP.

**Дальше:** clip по фактической геометрии при переносах, путь вместо rect, stencil там, где scissor не хватает.

---

### Фаза D — Группы прозрачности и blend modes

**Формат:** контейнер `GROUP` (или эквивалент): `opacity`, `blend-mode`, флаг **isolated** (и при необходимости knockout).

**PDF:** группы через Form XObject + `ExtGState` (`BM`, `ca`/`CA`, isolated group).  
**SVG:** вложенные `<g>` + `mix-blend-mode` / opacity (с учётом ограничений движков).  
**WGPU:** отдельный offscreen pass на группу, затем композит с выбранным blend.

**Критерий:** две вложенные группы с разными blend modes воспроизводятся в PDF; SVG и preview по возможности паритетны.

---

### Фаза E — Маски (SMask / luminosity и сложные маски)

**Формат:** ссылка на растровый или векторный ассет + режим маски; согласовать подмножество с PDF spec.

**PDF:** `SMask` в `ExtGState`, связка с Form/Image — максимальный объём работы среди фаз.  
**SVG:** `mask` + определения.  
**WGPU:** текстура маски, умножение альфы или отдельный pass.

**Критерий:** зафиксированное в спеке подмножество масок с тестами на golden или детерминированные PDF.

---

## Мелкие операторы и производительность

На уровне **Lura** допустимо много мелких семантических элементов; при **lowering в PDF** важны батчинг в одном content stream и минимизация лишних `q`/`Q` и смены `ExtGState`. Это отдельная задача оптимизации после корректности.

## Lura Graphics 1.0 (уточнение)

**Фактически закрыто в коде:** подмножество **B + clip-rect (C MVP)** без фазы **A** (кастомные шрифты). Имеет смысл зафиксировать в SPEC отдельно:

- атрибуты: `opacity`, `overflow: clip | hidden`;
- IR: перечисленные выше команды стека;
- известные ограничения (список, многостраничные блоки, без blend/SMask).

**Следующий шаг спецификации:** описать **Graphics 1.0** в [SPEC.md](SPEC.md) так, как оно уже ведёт себя в парсере и бэкендах; затем либо включить **фазу A** в «1.1», либо оставить 1.0 без встраиваемых шрифтов до готовности A.

**Следующий шаг кода (кратко):** фаза **A** (шрифты) или доводка B/C (list items, clip при переносе страниц), затем **D** (группы / blend).
